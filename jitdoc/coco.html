Coco is a small extension to get True C Coroutine semantics for Lua 5.1.

Coco is both available as a stand-alone release and integrated into LuaJIT.

The stand-alone release is a patchset against the » standard Lua distribution. There are no dependencies on LuaJIT. However LuaJIT depends on Coco to allow yielding for JIT compiled functions.

Features

True C coroutine semantics mean you can yield from a coroutine across a C call boundary and resume back to it.

Coco allows you to use a dedicated C stack for each coroutine. Resuming and yielding to/from a coroutine automatically switches C stacks, too.

In particular you can now:

  • Yield across all metamethods (not advised for __gc).
  • Yield across iterator functions (for x in func do).
  • Yield across callbacks (table.foreach(), dofile(), ...).
  • Yield across protected callbacks (pcall(), xpcall(), ...).
  • Yield from C functions and resume back to them.

Best of all, you don't need to change your Lua or C sources and still get the benefits. It's fully integrated into the Lua core, but tries to minimize the required changes.

Coco is still work-in-progress. It will likely change quite a bit in the next few releases due to some upcoming changes (e.g. mmap'ed C stacks or faster context switching with inline assembly).

More ...

The latest stand-alone release of Coco is a pre-release for Lua 5.1-work6. It will be updated shortly after the initial release of LuaJIT. Coco is already included in the LuaJIT download package.

Coco needs some machine-specific features — please have a look at the Portability Requirements. Coco also provides some upwards-compatible API Extensions for Lua.
